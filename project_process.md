# 프로젝트 진행과정

## 핵심 원칙

- 처음부터 거대한 서비스를 목표로 하지 않는다
- **내가 직접 불편함을 느끼는 작은 문제**에서 출발한다
- 아무도 안 쓰더라도 **내가 계속 사용할 서비스**를 만든다
- 서버 운영비 0원 구조를 유지해 방치해도 부담이 없게 한다

---

## 기술 스택

| 역할 | 도구 | 비용 |
|------|------|------|
| AI 코딩 | ChatGPT (Codex) / Claude | 유료 구독 |
| 버전 관리 | GitHub | 무료 |
| 배포 | Cloudflare Pages | 무료 |
| 로컬 확인 | VS Code + Live Server 확장 | 무료 |
| 도메인 | 선택 사항 | 연 ~2만원 |

> Cloudflare Pages는 HTML/CSS/JS 정적 파일만 지원한다. 게시판·회원가입 같은 동적 기능은 구현 불가.

---

## 진행 단계

### 1단계 — 아이디어 선정

- 일상에서 내가 직접 겪는 불편함을 찾는다
- 이미 비슷한 서비스가 있어도 괜찮다. 기능이 분산돼 있거나 불편하다면 만들 이유가 있다
- 정적 웹사이트로 해결 가능한 문제인지 확인한다 (계산기, 변환기, 편집기, 체크리스트 등)

### 2단계 — AI로 개발 (바이브 코딩)

1. 프로젝트 폴더 생성
2. Codex(ChatGPT) 실행
3. 프롬프트에 의도를 명확하게 작성
   - 원하는 서비스 설명
   - 정적 페이지(HTML/CSS/JS)로 만들어달라고 명시
   - 애드센스 심사를 진행할 것이니 심사 준비도 함께 해달라고 요청
   - 유지보수가 편리한 구조로 만들어달라고 요청
4. VS Code Live Server로 결과물 확인
5. 원하는 기능을 하나씩 추가 요청

### 3단계 — 배포

1. 완성된 파일을 GitHub 저장소에 올린다
2. Cloudflare Pages에서 GitHub 계정 연결
3. 자동 배포 설정 완료 → 서버 비용 0원으로 운영 시작

### 4단계 — 애드센스 심사 준비

AI가 만든 결과물을 그대로 제출하면 안 된다. 아래 세 가지를 직접 확인해야 한다.

**심사 통과 체크리스트**

- [ ] 실제로 작동하는 기능이 있는가 (스팸 사이트처럼 보이면 탈락)
- [ ] 페이지가 충분히 채워져 있는가 (기능 설명 / 사용 방법 / FAQ / 관련 팁)
- [ ] 기본 정책 페이지가 있는가 (개인정보처리방침 / 문의 / 사이트 소개)

준비가 끝나면 Codex에게 다음 프롬프트로 마지막 점검을 요청한다:
```
구글 애드센스 심사를 위한 준비가 충분히 됐는지 확인해줘.
사이트맵 추가 등 SEO 최적화도 함께 진행해줘.
```

심사 기간은 약 2주 소요.

### 5단계 — 홍보

**기본 (필수)**
- 구글 서치 콘솔에 사이트 등록
- 네이버 서치 어드바이저에 사이트 등록
- 검색 노출 확률을 높이는 가장 기본적인 작업

**커뮤니티 홍보**
- 내 서비스와 관련된 사람들이 모이는 커뮤니티를 찾는다
- 무턱대고 도배하면 역효과. **타겟이 맞는 곳에만** 홍보 게시글 작성
- 카카오톡 오픈 채팅방에서 활동하며 자연스럽게 소개

---

## 유지 전략

- 서버 운영비 0원 → 방치해도 부담 없음
- 내가 직접 쓰는 서비스 → 사용자가 없어도 동기 유지
- 도메인 비용(연 ~2만원)은 애드센스 수익으로 충당
